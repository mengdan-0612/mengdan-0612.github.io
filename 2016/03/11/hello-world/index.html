    • ##PHP安装

    下载安装包并解压

    安装之前先安装需要开的扩展（查看扩展参数，如./configure -help|grep mysql）

    yum install mysql mysql-devel
yum install gd
yum install mysql-server(mysql服务)

    配置 编译

    (想再次安装的时候可以用make clean清除之前的配置)

    ./configure --prefix=/usr/local/php --with-mysql=mysqlnd --enable-mysqlnd --with-gd --enable-gd-native-ttf --enable-gd-jis-conv --enable-fpm

    --enable-fpm表示已fpm运行php

    复制两个配置文件到安装后的目录

    cp /usr/local/src/php-5.6.17/php.ini-development ./bin/php.ini
cp etc/php-fpm.conf.default etc/php-fpm.conf

    开启php-fpm

    ./sbin/php-fpm

    重启：kill -HUP cat var/run/php-fpm.pid
    （php-fpm.pid的路径要在php-fpm.conf里配置）

    配置nginx

    location ~ \.php$ {
    root  html;#把根目录定位到html
    fastcgi_pass   127.0.0.1:9000;#把请求转发给9000端口的php进程
    fastcgi_index  index.php;#默认文件是index.php
    fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;#对应文件
    include        fastcgi_params;#加载参数文件
}

    把根目录定位到html,把请求转发给9000端口的php进程，让php进程处理，当前的脚本是$document_root$fastcgi_script_name
    （PHP会去找这个脚本并处理,所以脚本的位置要指对）

    开启openssl扩展

    进入PHP的安装文件下的openssl扩展模块目录

    cd php-5.2.14/ext/openssl/
/var/www/php5/bin/phpize

    这里为你自己的phpize路径，如果找不到，使用whereis phpize查找
    执行后，发现错误 无法找到config.m4 ，config0.m4就是config.m4直接重命名或者复制一份

    mv config0.m4 config.m4
/var/www/php5/bin/phpize

    编译

    ./configure --with-openssl --with-php-config=/var/www/php5/bin/php-config
make
make install

    配置php.ini

    安装完成后，会返回一个.so文件（openssl.so）的目录。
    .so 文件目录
    /usr/local/src/php-5.6.17/ext/openssl/modules

    在此目录下把openssl.so 文件拷贝到你在php.ini 中指定的 extension_dir 下
    （在php.ini文件中查找：extension_dir =），我这里的目录是 var/www/php5/lib/php/extensions
    编辑php.ini文件，在文件最后添加
    extension=openssl.so

    重启Apache即可

    /usr/local/apache2/bin/apachectl restart
